# 科研记录

这是“科研记录”的 PySide6 源码版。软件默认运行目录为 `E:\科研记录`，业务数据只保存在本机运行目录，不提交到 GitHub。

## 数据存储

业务数据不再使用 JSON 文件保存。旧 JSON 仅允许在首次启动时作为兼容数据源只读迁移；迁移完成后，软件不再生成或写入用于保存任务、标签、文章、图片、学习记录、每日心得或登录信息的 JSON 文件。

正式数据位置：

```text
E:\科研记录\
├─ 数据\
│  ├─ images\
│  │  ├─ 框架图\
│  │  │  └─ 20260522_103000_框架图_001.png
│  │  └─ 论文截图\
│  └─ research_record.db
└─ 科研记录.exe
```

- 任务、标签、文章、图片元数据、学习记录、每日心得、登录信息和设置保存到 SQLite：`E:\科研记录\数据\research_record.db`。
- 图片保存为真实图片文件，并按标签文件夹归类：`E:\科研记录\数据\images\标签名\文件名.png`。
- SQLite 只保存图片相对路径和元数据，例如 `images/框架图/20260522_103000_框架图_001.png`。
- GitHub 仓库只保存源码、依赖说明和项目文档，不保存用户真实数据、图片、数据库、exe、zip、缓存和备份文件。

## SQLite 表

主要表包括：

- `users`：本地账号、盐值和密码哈希。
- `settings`：主题、背景、登录记忆等简单设置。
- `tasks`：任务内容、优先级、截止时间、完成状态和任务日期。
- `study_sessions`：学习时长记录。
- `daily_notes`：每日心得。
- `articles`：论文题目、DOI / Links、原文链接和文章备注。
- `tags`：图片标签名称、颜色和时间戳。
- `image_records`：文章下的图片相对路径、图片标签、图片备注和关联元数据。

## 功能

- 今日学习打卡、今日任务、四象限任务归类、每日心得。
- 今日任务支持任务内容、优先级、截止时间和完成状态，逾期未完成任务会提示。
- 图谱页按文章管理多张图片，支持选择图片、拖拽图片和 Ctrl+V 粘贴剪贴板图片。
- 图谱页通过统一的“检索并跳转”按钮打开 DOI / Links，题目检索使用 Crossref 公开接口，不爬取 Google Scholar 或知网页面。
- 统计页提供日历、选中日期详情、学习时长折线图和任务数量条形图。

## 运行

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
.\.venv\Scripts\python -m research_record
```
